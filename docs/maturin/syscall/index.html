系统调用实现

目前的系统调用规范参照比赛所提供的类似 Linux 系统调用实现。

有一些注释的系统调用名，那些是发现应用需要，但尚未实现暂时跳过的系统调用

这两种调用间比较容易混淆的区别是，比赛测例是用 C 写的，大部分数组都是 4 Byte， 而 rCore 使用 rust，usize/isize 一般是 8 Byte。 这导致一些传入地址(非字符串,字符串大家都是统一的 1Byte 类型)的大小有问题， 如 sys_pipe() 在测例环境下需要将输入作为 *mut u32 而不是 *mut usize
